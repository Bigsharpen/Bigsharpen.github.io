#NODE环境

##什么是node

  • 不是js文件，也不是js框架
  • 而是Server side Javascript runtime，服务端的一个js运行时
  • 目前我们的js是运行在浏览器内核中
  • Php是一门脚本语言，也是一个运行环境
  • node是一个轻内核的东西
  • 为什么Node选中了js，
    +javascript组成：ECMAscript JS-ES BOM（浏览器对象模型） DOM（文件对象模型）
    • 浏览器组成 ：js引擎和渲染引擎
  • node是用来执行js的运行环境（用的是Crome v8引擎）
  • nvm（node version manager）node版本管理工具
  • npm（node Package Manager）node应用程序依赖包的管理工具，执行安装卸载之类的操作
    +淘宝数据源:npm.taobao.org
    • 常见的包都有循环依赖的功能，会自动安装依赖的包
    • 安装包的命令：npm install 包名
    • 会自动安装稳定版本
  • Bower（Web应用程序依赖管理工具）
    • 安装Bower npm install -g bower

      ##复习
      ###node是什么
  • node 是一个js的运行环境
  • 用于开发web应用程序
  • 很多前端开发工具都是基于node平台
    ###nvm是什么
  • node version manager（node版本管理工具）
  • 因为node版本比较多，很多时候我们可能依赖多个版本，并要求自由切换
  • 使用：
    • 切换版本：nvm use 版本号
    • 安装： nvm install 版本号
    • 卸载： nvm uninstall 版本号
    • 已安装列表：nvm list
      ###什么是npm
  • node package manager（node包管理工具）
  • node是轻内核的，没自带什么功能，对各种包的依赖很大
  • nom管理包非常方便，我们只需要记住用哪个包就可以了
  • 使用：
    • npm install 包名 -g 全局安装
      -安装的是工具的时候，需要共享，每个地方都能用
    • npm install 包名 安装到项目本地
      • 必须要联网
      • 在项目文件夹打开cmd
      • 会在项目文件夹下载一个node_modules目录
      • 需要用一个package.json文件来记录依赖的包（npm init产生这个文件）
      • 以后安装包的时候命令后面加上 –save,会自动将包的名字和版本记录到json文件里面
      • 加上–save-dev下载开发依赖包的时候，加上–save下载项目依赖包的时候
      • 项目依赖（项目运行必须的如bootstrap，被记录在dependencise）和项目开发依赖（只用来帮助开发的工具如gulp，被记录在devdependencise）
        ###什么是Bower
  • 一个和npm差不多的包管理工具
  • 一般用于管理Web页面开发时使用的包如Jquery，bootstrap
    ###fis百度开发的和gulp差不多的一个工具

##Git（源代码管理工具）

  • 在一个项目中由开发人员写的都是源代码
  • 管理源代码，让代码可以被追溯
  • 主要记录每次变更了什么，谁主导了这次变化
  • Git是linux之父当年为了维护linux的源代码写的一个工具
    ###Git命令
  • git help 最重要的命令
  • git init 初始化一个本地仓库
  • git status -s 用于查看本地仓储的状态
  • git add 可以将一个没有被跟踪的文件添加到跟踪列表
  • 添加本地GIT忽略清单文件
    在代码库的根目录添加一个名为.gitignore文件用于说明忽略那些文件
  • git commit -m ‘提交日志’ 将本地的变化提交到本地的仓库文件夹，每一次提交必须要有提交日志，一般在有了一个小单元的变化之后再提交
  • git diff 可以用于对比当前状态和版本库状态的对比
  • git log 查看提交日志
  • git reset –hard
    ##Github
  • 是一个Git服务提供商

###Git分支

  • 就是同一个项目的不同版本
    ##在Git上面展示页面（add-commit-push）
    ###地址：bigsharpen.github.io/项目名
  1. 在项目根目录新建一个仓库git init
  2. 将文件托管到Git git add –all
  3. 将文件提交到Git git commit -m ‘说明’
  4. 在github上创建一个仓库 respositories 并获取SSH
  5. 将本地文件仓库同步到github仓库git remote add origin SSH
  6. 将本地文件push到git主分支 git push -u origin master
  7. 创建一个名为gh-pages的分支git branch gh-pages
  8. 切换到gh-pages分支
  9. 将文件push到gh-pages分支git push -u origin gh-pages

##函数（方法）

  • 一种工具——>封装性
  • 只需要知道怎么使用
  • 我们学习的所有函数都是某个对象的方法
    ##对象（字符串，日期，Math，数组，正则）
  • 工具包：把相似的工具（功能）放在一起，便于使用
  • 包含两种东西：函数，属性
  • 万物皆对象
  • 整个js都是面向对象的
    ##对象以及实例化
  • 对象是抽象，实例是具体
  • 手机是抽象，不同型号的手机产品是实例
  • 我们只能使用实例，不能使用抽象化的东西
  • 要使用对象要先实例化：var 实例 = new 对象（）；
    ##面向对象编程的步骤：
  1. 思考需要哪几个对象
  2. 分析属性和方法，先把空壳（接口）写出来
  3. 实现工具：写代码封装成函数和对象
  4. 使用工具
